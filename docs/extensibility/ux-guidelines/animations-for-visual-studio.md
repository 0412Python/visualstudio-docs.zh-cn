# Visual Studio 的动画效果
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## 动画基础知识  
  
### 在 Visual Studio 中的动画最佳做法  
 请按照这些规则，以确保在整个 Visual Studio IDE 一致且用户友好的动画样式。  
  
-   **将仔细选择。**限制为特定目的的动画。  
  
-   **计时和速度非常重要，** 以确保转换觉得快速和自然:  
  
    -   半秒 \(500 毫秒\) 内进行完整的动画的转换。  
  
    -   会经常发生的动画需要快速，它们不会中断用户的工作流。  
  
    -   动画不应是快速，或不和谐很难以理解，但不是太短，它是一个耐心以便完成转换。  
  
    -   使用变量计时来强调重要性。 例如，导航时通过在类图上的项的序列，经过转换的各项之间的速度，则减慢速度以专注于重要的项。  
  
-   **通过逐步非线性缓动** 从到另一种状态，为提供的平静和自然移动的意义上  
  
-   如果可能， **悬停时的使用细微动画** 以指示在鼠标交互元素。  
  
-   如果您依赖于很大程度上动画在您的功能然后 **提供一种方法将其关闭** 本地 \(对于所有功能\) 中的一个选项为 **工具 \> 选项** 对话框。  
  
-   **只能有一个动画应发生一次** ，传达只是一条信息。  
  
-   **细微差异非常重要。**在大多数情况下动画不必将要求用户采取行动，以提供其用途。 细微变化计时、 排序、 和的行为可能会显著影响角度看，并且可以使有效和无效动画之间的差异。  
  
-   当使用动画，提醒大家注意到某件事情， **确保，因此有必要中断用户**的思路。  
  
-   **时显示进度或状态** 通过动画:  
  
    -   停止不提前基础进程时显示进度移动。  
  
    -   将不确定的进程与确定的进程区分开来。  
  
    -   确保动画具有可识别的完成和失败状态。  
  
    -   最小化效果动画，显示状态并确保它们具有真正的价值所提供的实际应用中的附加信息的使用。 示例包括暂时性的状态更改和紧急情况  
  
#### 不这样做:  
  
-   使用小移动 \(运动中较小需求量\)，而偏向于淡出并更改通过移动对象。  
  
-   使用动画都通过屏幕空间较大区域。 而不考虑大小，这种样式是动画的让人分散注意力给用户。  
  
-   使用与当前侧重于用户的对象或与交互不相关的动画。  
  
-   使用需要用户交互才能重置状态，例如强制用户响应以使其停止闪烁闪烁通知的动画。 以任何方式与它们交互应当已经足够关闭它们。  
  
 有关这些最佳实践的应用程序的详细信息，请参阅 [动画模式](../../extensibility/ux-guidelines/animations-for-visual-studio.md#BKMK_AnimationPatterns)。  
  
### 动画度量值  
  
-   系统应显示用户在与手势小于 10 毫秒做出响应。  
  
-   动画的过渡考虑不应超过 500 毫秒才能完成。  
  
-   一种方法来弥补需要较长的时间的转换是分离到两个部分:例如，动画的第一部分可能是到容器 \(最多 500 毫秒\) 后跟内容淡入淡出空内容容器 \(最多 500 毫秒\)。  
  
-   对于可计算的加载时间，决定进度指示器 \(%完成进度指示符\) 是首选。  
  
-   对于无法计算的加载时间，例如游标或嵌入的旋转动画 \(加载或工作指示器\) 的忙碌状态指示器是适当。  
  
### 作为 communicator 动画  
 在 Visual Studio UI 中，动画函数仅作为一种通信工具。  要使用它进行通信的各种信息，如 UI，则发生了结构变化例如，菜单打开或关闭时。 动画可帮助实现可视化效果的复杂系统中，如安装进度可视化时间相关的行为，或用来吸引注意力与警报和通知。  
  
 通常，此 UI 动画功能在四种方法: 可视化、 吸引注意力，模拟，并指示响应时间\/进度。  
  
#### 实现可视化效果  
 动画可以强调三维对象的性质，并使用户更轻松地直观显示其空间的结构。 若要实现此目的，动画可能需要以旋转一整圈中的对象缓慢，并且反复打开或将对象更接近并略有增加其大小，以强调滚动更新或焦点。  
  
 尽管三维对象可以移动与用户控件 \(以编程方式或手动安装\)，在设计器应事先确定如何为最佳设置动画效果提供了对该对象的最佳了解移动。 此编程动画可以然后可由用户激活通过将光标放在悬停在对象上，而用户控制移动需要使用户能了解如何操作对象。 将移动限制为单个轴或一次; 的方向要么缩放、 旋转、，或转换，但不同时执行多个。  
  
 可视化效果类别包括的数据、 关系、 状态方面结构、 序列和时间。  
  
##### 数据  
 说明了复杂和变量信息:  
  
-   如图表和图形信息可视化对象间移动  
  
-   通过一系列单步执行、 指导的教程中和分页  
  
-   调用的详细信息、 指向，并突出显示特定信息  
  
-   覆盖的详细信息以及在具有焦点的元素之上的其他信息  
  
-   到另一种结构或组织的表示形式从变形  
  
-   随着使用时间滑块、 作业乘坐车轮和传输控件 \(播放、 停止和暂停\) 的时间的推移，表示更改。  
  
##### 关系  
  
-   阐释如何项关联到另一个接口或哪些项与给定项相关。  
  
-   显示层次结构和父\-子链接或同级关系  
  
-   一个元素生成另一个  
  
-   一个元素到另一个元素降至最低  
  
-   已接入到另一个元素  
  
##### 状态  
  
-   内容更新。  
  
-   用户焦点和选择内容  
  
-   进度  
  
-   错误  
  
##### 结构  
  
-   透视的一个节点上的结构  
  
-   重定向  
  
-   最小化和最大化，或展开和折叠  
  
##### 序列  
  
-   幻灯片放映序列  
  
-   再逐页浏览图片  
  
##### 时间  
  
-   显示随时间、 时间间隔和截屏视频  
  
-   移到他罪有应得，、 撤消和重做  
  
-   还原历史记录状态  
  
#### 引起关注  
 如果目标是使用户的注意力集中到多个外的单一元素，或更新的信息向用户发出警报，则动画可能比较合适。 例如，你的应用程序的起始页可能会采用滑入到位，在页面加载之后的入门按钮。  
  
 通常，在屏幕上的最后一个移动元素能够吸引用户的注意力。  在一系列的动画元素，将用户的注意力将按照的最后一个移动对象。  
  
##### 发出警报  
  
-   向用户发出警报，获取关注，显示进度  
  
-   显示的执行某些操作被正确或错误或显示进度或进行更改  
  
-   在一个任务，例如查找的详细联机信息也不需要学习有关当前任务过程中提示用户  
  
##### 通知  
  
-   有关错误条件向用户发出警报  
  
-   打断用户若要查看其是否希望以处理其他事项  
  
-   轻轻地通知过程已完成或更改，例如，完成下载后的用户。  
  
#### 模拟  
 此类别包含物质至上和维数。  
  
-   说明各个对象的来源或他们转到  
  
-   展开和折叠或打开和关闭  
  
-   平移、 滚动和页面启用  
  
-   堆栈和 z 顺序  
  
-   轮播和可折叠面板  
  
-   翻转和旋转用户界面  
  
#### 响应和进度指示器  
 进度指示器有几个显著优点:  
  
-   两个确定的和不确定的进度指示器再次向确认的用户，系统未损坏，并着手解决的问题。  
  
-   确定指示器赋予的用户一种多久完成该操作时的进展情况，以及获取更接近于完成的感觉。  
  
##  <a name="BKMK_AnimationPatterns"></a> 动画模式  
  
### 概述  
 在 Visual Studio 中的动画是为了服务于特定函数并且不会妨碍用户工作效率。 若要遵循以包括的常规动画特征:  
  
-   小型和非介入式  
  
-   自然而真实  
  
-   难以捉摸也最 subdued  
  
-   快速、 高效地  
  
-   放松、 不名匆忙之中  
  
 下图显示建议在 Visual Studio 中使用的动画样式。 没有动画和精致的动画效果如淡入\/淡出最常使用。 有限的应用程序的移动动画如扩展和收缩，X 和 Y 位置更改和旋转。  
  
 ![Recommended animation styles for Visual Studio](../../extensibility/ux-guidelines/media/1202-a_vsanimstyles.png "1202\-a\_VSAnimStyles")  
  
 **Visual Studio 的建议动画样式**  
  
#### 显示和隐藏  
 与此模式中，元素将从切换可见到扩展的视图，而返回过渡动画:  
  
 ![Appear&#47;Disappear animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-b_appearanddisappear.png "1202\-b\_AppearAndDisappear")  
  
##### 正确用法  
 需要立即显示或消失，以便用户不将注意力集中也不阻碍，因此全新用户界面元素。 此外，缓慢移动动画感觉像性能拖动、 使用出现和消失样式不会发生这种情况。  
  
##### 用法不正确  
 在其中用户界面出现突然用户完全不知道发生了什么情况，并添加动画将帮助解决上下文理解的情况。  
  
##### 动画属性  
 时间延迟通常为 0 秒。  
  
##### 示例  
  
-   自动隐藏工具窗口  
  
-   键盘激活编辑器用户界面，例如智能感知和参数帮助  
  
-   展开折叠代码区域  
  
#### 淡入和淡出  
 使用此模式中，UI 元素从托管代码转换不可见 \(0%不透明\) 到可见 \(100%不透明\) 或执行相反:  
  
 ![Fade&#45;in&#47;Fade&#45;out animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-c_fadeinfadeout.png "1202\-c\_FadeInFadeOut")  
  
##### 正确用法  
 这是最常用建议 UI 动画。 它是一种细微的效果，而不会中断流添加感兴趣。 在某些情况下，用户可能没有意识到到那里一个动画，并只是感觉平滑、 流动的 UI 系统。  
  
##### 动画属性  
  
-   启动不透明度: 淡入的 0%、 淡出的 100%  
  
-   终点不透明度: 淡入的 100%、 淡出的 0%  
  
-   持续时间: 200 毫秒独立，100 毫秒作为组合动画序列的一部分时使用  
  
-   缓动样式: 正弦 InOut  
  
##### 示例  
  
-   自动隐藏工具窗口  
  
-   菜单打开和关闭  
  
-   前景色和背景选项卡上转换  
  
#### 从 A 到 B 的颜色混合  
 UI 元素组装在此模式下，它更改颜色 A 为颜色 b:  
  
 ![Color blend animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-d_colorblend.png "1202\-d\_ColorBlend")  
  
##### 正确用法  
 与用户界面元素从一个上下文或状态到另一个更改颜色时动画转换。  
  
##### 动画属性  
  
-   开始颜色: 特定于用户界面  
  
-   结束颜色: 特定于用户界面  
  
-   持续时间: 200 毫秒独立，100 毫秒作为组合动画序列的一部分时使用  
  
-   缓动样式: 正弦 InOut  
  
##### 示例  
  
-   文档窗口中状态转换 \(处于活动状态，最后一处于活动状态，以及处于非活动状态\)  
  
-   工具窗口状态过渡 \(变得精准而失去焦点\)  
  
#### 扩展和收缩  
 使用此模式中，UI 元素会在 X、 Y 或两个方向中展开:  
  
 ![Expand&#47;Contract animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-e_expandcontract.png "1202\-e\_ExpandContract")  
  
##### 正确用法  
 与用户界面元素到另一个上下文从更改大小时动画转换。  
  
##### 动画属性  
  
-   X 小数位数: %或特定的尺寸 \(以像素为单位\)  
  
-   Y 小数位数: %或特定的尺寸 \(以像素为单位\)  
  
-   定位点位置: 通常 \(用于从左到右的语言\) 的左上角或右上角 \(对于从右到左的语言\)  
  
-   持续时间: 200 毫秒独立，100 毫秒作为组合动画序列的一部分时使用  
  
##### 示例  
  
-   体系结构资源管理器面板展开和折叠  
  
-   起始的页项目展开和折叠  
  
#### X Y 位置更改  
 使用此模式中，UI 元素将更改和 \/ 或其 X 或 Y 位置:  
  
 ![X&#47;Y position change animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-f_xypositionchange.png "1202\-f\_XYPositionChange")  
  
##### 正确用法  
 与用户界面元素到另一个上下文从更改位置时动画转换。  
  
##### 动画属性  
  
-   启动 X 和 Y 位置: 特定于用户界面  
  
-   结束 X 和 Y 位置: 特定于用户界面  
  
-   运动路径: 无  
  
-   持续时间: 200 毫秒独立，100 毫秒作为组合动画序列的一部分时使用  
  
-   缓动样式: 正弦 InOut  
  
##### 示例  
 选项卡重新排序  
  
#### 旋转  
 使用此模式中，将旋转的 UI 元素:  
  
 ![Rotate animation in Visual Studio](../../extensibility/ux-guidelines/media/1202-g_rotate.png "1202\-g\_Rotate")  
  
##### 正确用法  
 仅对不确定旋转进度指示器。  
  
##### 动画属性  
  
-   旋转的度数: 360  
  
-   旋转中心: 中间的对象  
  
-   持续时间: 连续  
  
##### 示例  
 不确定的进度指示器 \(旋转\)
