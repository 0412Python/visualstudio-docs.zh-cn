# 评估 Tools for Visual Studio
[!INCLUDE[vs2017banner](../../code-quality/includes/vs2017banner.md)]

## 针对 Visual Studio 的耕耘所得到核对清单  
 使用此清单来评估 visual 和交互的详细信息的用户体验质量。  
  
### 概述  
  
-   验证所有命令都产生的反馈，以告诉用户其命令已执行完毕。  
  
-   验证所有用户界面元素和控件都是可见的所有主题和高对比度模式。  
  
-   验证的非活动状态并处于活动状态的所选内容始终区别在于，在标准和高对比度模式。  
  
-   验证焦点始终可见和明显。  
  
### 性能  
  
-   验证某些类型的"忙"指示器显示是否一个命令将超过一秒才能完成。  
  
-   如果命令需要多个 10 秒钟才能完成，显式进度栏中，或者验证确定的 \(首选\) 或不确定的将显示。  
  
### UI 文本  
  
-   验证所有标签都的句子或为首字母大写，并且没有文本是完全小写。  
  
    ||更正|不正确|  
    |-|--------|---------|  
    |**命令文本 \(全部\)**|句子大小写:<br /><br /> **目录名称:**|目录名称:|  
    |**按钮文本 \(客户端\)**|为首字母大写:<br /><br /> **\[设为默认值\]**|SET AS DEFAULT|  
    |**按钮文本 \(联机\)**|句子大小写:<br /><br /> **\[设为默认值\]**||  
  
-   验证所有标签，除组标题和按钮，以冒号结束，并且表达式与其配对。  
  
-   验证按钮、 命令和启动用户界面以捕获用户输入的命令链接以省略号结尾 **\[...\]**。  
  
     示例：  
  
    -   **高级 …** 对话框上的按钮。  
  
    -   在工具菜单下的命令选项 \(**工具 \> 选项**\) 应获取省略号，因为启动对话框本身为命令的目的。  
  
-   验证 UI 包含不缩写，除了行业标准术语。 例如，HTML 和 TCP\/IP 都不需要拼写出来，但应在 OOM \(内存不足\) 和 PII \(个人身份信息\)。  
  
### 键盘访问  
  
-   验证完成键盘与每个任务的方法。 通常这一点通过为每个控件的键盘访问，但对于某些极为直观的区域，如转到代码视图变通的办法是可接受。  
  
-   请验证您可以通过控件按逻辑顺序 \(从左到右和顶部到底部\) 选项卡。 当大多数控件的最佳做法，并非所有控件都需要这种方法。 例如，验证该控件是具有单个制表位的组中的单选按钮。  
  
-   验证所有控件都具有标签以及每个标签具有助记键 \(例外情况包括某些程序时应遵循选项卡中的带标签的控件的不带标签控件\)。  
  
-   验证没有助记键冲突。  
  
### 字体  
  
-   验证所有字体 \(字体、 大小、 颜色\) 统一使用和维护层次结构。  
  
-   验证所有 UI 元素都使用环境字体服务。 \(请参阅 [字体和格式对 Visual Studio](../../extensibility/ux-guidelines/fonts-and-formatting-for-visual-studio.md)\)  
  
     若要检查是否正在使用该服务，请转到 **工具 \> 选项 \> 字体和颜色**。 在设置下拉列表中，选择环境字体，将字体更改为其他开发不同 \(如 Harrington 或漫画 San\)，并将大小设置为 12 磅。 然后单击确定。 您可能需要重新启动 IDE 中，但大多数 UI 将会立即更改。 避免拿字体更改甚至在重新启动的区域不使用环境字体。  
  
-   验证服务 \(例如，放大或缩小加粗文本\) 的派生类的字体保留其大小和环境字体大小发生更改时，相对于"normal"的文本格式。  
  
-   验证由于放大字体不剪辑纠正错误。 获取剪辑的字体可能都是固定的高度控件或固定的高度容器的结果。  
  
### 对话框  
  
-   验证对话框的标题启动它的命令相同。  
  
-   验证所有的标准控件是否与操作系统一致: 背景色是标准，因此没有控件应具有一种使它们看起来不同于标准控件特殊 re 模板化方式。  
  
-   验证窗体中的边距应为 12 像素，并且应显示统一和一致。  
  
-   验证对话框显示在 IDE 外壳程序或生成它们的窗口内居中。  
  
-   当很有用时，对话框应该是可调整大小。 可调整大小的对话框，请验证，在调整大小时，相应的控件必须调整大小而对话框中的其他部分保持不变。  
  
-   验证可调整大小对话框保留任何用户调整大小 \(大小、 位置、 扩展的对话框控件等\)。  
  
-   验证在标题栏中有无图标。  
  
-   验证没有最小化和最大化按钮的标题栏中。  
  
#### 对话框操作按钮 \(仅 VS 客户端\)  
  
-   验证操作按钮是否按以下顺序: **确定**, ，**取消**, ，**应用**。  
  
-   验证 **确定** 和 **取消** 按钮是标准大小: 75 x 23 像素为单位。  
  
-   验证 **确定** 和 **取消** 按钮是而不考虑字符串长度的大小相等。  
  
-   如果上一个操作按钮的标签要求要比标准更广的按钮，确认相应 **取消** 按钮是大小相等。  
  
-   验证有 6 个像素填充之间的按钮和关联的控件。  
  
-   验证 **确定** 和 **取消** 按钮没有助记键 \(访问键加带下划线的字母定义\)。  
  
-   验证该一个按钮 \(通常 **确定**\) 默认情况下具有焦点。  
  
-   验证 **Esc** 取消对话框  
  
-   验证 **Enter** 如果焦点不在处理输入控件中执行的默认按钮。  
  
-   验证 **确定** 和 **取消** 按钮在右下角的对话框中的位置。 在极少数例外情况是适用于它们会垂直堆叠在一起的右上角。  
  
-   验证使用垂直配置了只有其他按钮都位于对话框中为水平对齐方式。  
  
### 控制标准  
  
#### 常规  
  
-   验证，如果可能，有一些很好的默认值以加快用户交互并指导用户向安全或常见结果。  
  
-   验证，以便用户知道将发生什么情况取决于先前体验的标准控件的行为方式相同。  
  
#### Label 控件  
  
-   验证每个控件具有的标签，每个标签以可视方式配合 \(通常在 4\-6 像素范围内\) 其控件更接近于其他控件比其相应的控制。  
  
-   验证标签位于冲洗与控件左边缘，如果定位上面左右，使其标签的基线对齐与输入文本的基线，如果定位到左侧的水平，居中位置。  
  
-   验证是否多个堆积的标签和输入的控件控件的左侧定位的标签左对齐，并且从该对话框中，边缘等差永远不会刷新右和从控件等差。 除非他们需要增加额外的空间，以指示分组，对应均匀分布。  
  
#### 输入的控件 \(文本框和组合框\)  
  
-   验证在使用默认环境字体时，文本框、 组合框和按钮的显示高度所有 23 像素。  
  
-   如果使用提示文本，则验证将颜色设置 `Environment.ControlEditHintText` 使用颜色服务。  
  
-   如果此字段为必填的字段，必须在这种标识，请验证:  
  
    -   背景设置为 `Environment.ControlEditRequiredBackground` 和前景色设置为 `Environment.ControlEditRequiredHintText`  
  
    -   是否显示为控件中的提示文本 **"\< Required \>"**  
  
#### 按钮控件  
  
-   验证按钮是最小大小为 75 x 23 像素，除非容纳较长的文本。  
  
-   验证按钮具有左边框和右 3\-5 像素，以及填充内容的边距。  
  
-   它是可接受的小正方形按钮用于仅省略号 **\[...\]** 上而不是 **\[浏览...\]** 按钮 \(或类似的功能\)。 如果使用，请验证按钮为 23 x 23 的大小。  
  
-   如果有多个 **\[浏览...\]** 按钮在对话框中，然后验证缩写的版本 \(仅限省略号 **\[...\]**\) 应用于所有。  
  
-   验证该省略号 **\[...\]** 按钮没有助记键。 当焦点在其旁边的输入控件上时，一个选项卡应将焦点移到省略号按钮。  
  
-   验证按钮、 命令和启动辅助捕获更多用户输入的用户界面的命令链接必须以省略号结尾 **\[...\]**。  
  
#### 超链接  
  
-   验证超链接控件永远不会闪烁红色活动时。 这是颜色服务未使用指示器  
  
-   请验证使用的 VS 颜色:  
  
    -   `Environment.ControlLinkText`  
  
    -   `Environment.ControlLinkTextHover`  
  
    -   `Environment.ControlLinkTextPressed`  
  
-   验证显示有没有下划线蓝色除非段落中嵌入超链接。  
  
#### 复选框  
  
-   如果复选框将没有多行文本，请验证框中将使用第一行文本，不在所有行的垂直居中对齐。  
  
-   验证，复选框始终表示二进制可供选择，不要将用户定位文件或打开新窗口或页面。  
  
-   如果复选框显示相关的输入控件的选项，请验证其定位冲洗向左、 非常紧密，以指示其关系的控制之下。  
  
-   验证的复选框是 **永远不会** 作为一种方法用于启用的对话框或页面的全部内容。  
  
#### 分组框  
  
-   验证一个对话框，不包含其中一个包含对话框中的全部内容的单个组框。  
  
-   验证在每个组中至少两个控件。  
  
-   很少应该有两个以上的分组框上一个对话框。  
  
-   验证有任何嵌套的组框。  
  
### 图标  
  
-   验证图标深色主题中显示正确倒排。  
  
-   验证所有图标，所都依据的核心概念。  
  
-   验证每个图标是明确的、 可轻松地识别，并且不包含两个以上概念 \(无状态修饰符\/语言\)。  
  
-   验证该基图标将显示居中的空间内。  
  
-   验证所有图标都显示在高对比度模式清晰显示。  
  
-   验证使用任何颜色对应于颜色使用标准。  
  
-   验证不有任何光晕 \(边框\) 图标周围。 \(如果存在，halo 应匹配相邻的用户界面的背景色\)。  
  
### 支持触控的用户界面  
  
-   验证交互式控件是否足够大，无法轻松地可触摸 – 最小 **23 x 23 像素** 的大小  
  
-   验证最常用的控件是否至少 **40 x 40 像素** 的大小。  
  
-   验证是否至少具有交互式控件 **的间距的 5 个像素** 它们之间
