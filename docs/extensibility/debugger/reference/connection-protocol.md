# <a name="connectionprotocol"></a>CONNECTION_PROTOCOL
指示正在使用调试服务器和调试程序包 (DE) 之间进行通信的协议。  
  
## <a name="syntax"></a>语法  
  
```cpp  
typedef enum tagCONNECTION_PROTOCOL {  
   CONNECTION_NONE    = 0,  
   CONNECTION_UNKNOWN = 1,  
   CONNECTION_LOCAL   = 2,  
   CONNECTION_PIPE    = 3,  
   CONNECTION_TCPIP   = 4,  
   CONNECTION_HTTP    = 5,  
   CONNECTION_OTHER   = 6  
} CONNECTION_PROTOCOL;  
```  
  
```c#  
public enum CONNECTION_PROTOCOL {  
   CONNECTION_NONE    = 0,  
   CONNECTION_UNKNOWN = 1,  
   CONNECTION_LOCAL   = 2,  
   CONNECTION_PIPE    = 3,  
   CONNECTION_TCPIP   = 4,  
   CONNECTION_HTTP    = 5,  
   CONNECTION_OTHER   = 6  
};  
```  
  
#### <a name="parameters"></a>参数  
 CONNECTION_NONE  
 对服务器不进行了任何连接。  
  
 CONNECTION_UNKNOWN  
 已建立连接，但它属于未知类型。  
  
 CONNECTION_LOCAL  
 连接是连接到本地服务器。  
  
 CONNECTION_PIPE  
 连接是通过命名管道。  
  
 CONNECTION_TCPIP  
 连接使用 TCP/IP。  
  
 CONNECTION_HTTP  
 将使用 HTTP 连接 （通过 Web 服务器上）。  
  
 CONNECTION_OTHER  
 其他类型的连接建立 （此值当前未使用）。  
  
## <a name="remarks"></a>备注  
 这些值从返回[GetConnectionProtocol](../../../extensibility/debugger/reference/idebugcoreserver3-getconnectionprotocol.md)方法。
