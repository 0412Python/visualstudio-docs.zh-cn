# <a name="c26415-smartptrnotneeded"></a>C26415 SMART_PTR_NOT_NEEDED
"只对访问包含指针使用智能指针参数。 改用 T * 或 &。"

**C + + Core Guidelines**: R.30： 将智能指针作为参数仅对显式 express 生存期语义

使用智能指针类型将数据传递给一个函数表示在目标函数需要管理包含的对象的生存期。 但是，如果该函数仅使用智能指针来访问包含的对象，永远不会实际调用任何代码，这可能会导致其解除分配 （永远不会影响其生命周期），通常没有必要使情况更加复杂和智能指针的接口。 普通指针或引用包含的对象应为首选。

## <a name="remarks"></a>备注
- 此检查是涵盖大多数的情况下还将导致 C26410、 C26415、 C26417 和 C26418。 它是更好地清理 SMART_PTR_NOT_NEEDED 第一次，然后切换到共享或唯一指针的边缘情况。 为更有针对性地清理此警告可以禁用。
  - 除了标准 std::unqiue_pointer 和 std::shared_pointer 模板，此检查认识到用户定义类型都有可能应是智能指针。 这种类型需要定义以下操作：
  - 重载取消引用或成员访问运算符，是公开的未标记为已删除;
  - 公共的析构函数这是既不删除也默认设置。 这包括显式定义为空的析构函数。
  - 可能会影响包含对象的生存期的操作的解释是非常广泛，包括：
  - 接受与非常量的智能指针; 的指针或引用参数的任何函数
  - 复制或移动构造函数或赋值运算符;
  - 非常量函数。
    ## <a name="example"></a>示例
    繁琐的生存期管理

```cpp
bool set_initial_message(
            const std::unique_ptr<message> &m) // C26415, also C26410 NO_REF_TO_CONST_UNIQUE_PTR
{
    if (!m || initial_message_)
        return false;

    initial_message_.reset(m.get());
    return true;
}

void pass_message(const message_info &info)
{
    auto m = std::make_unique<message>(info);
    const auto release = set_initial_message(m);
    // ...
    if (release)
        m.release();
}
```

## <a name="example"></a>示例
繁琐的生存期管理-重写

```cpp
void set_initial_message(std::shared_ptr<message> m) noexcept
{
    if (m && !initial_message_)
        initial_message_ = std::move(m);
}

void pass_message(const message_info &info)
{
    auto m = std::make_shared<message>(info);
    set_initial_message(m);
    // ...
}
```
