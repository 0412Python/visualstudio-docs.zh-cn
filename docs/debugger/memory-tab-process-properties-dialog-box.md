# “进程属性”对话框 -&gt;“内存”选项卡
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

使用“内存”选项卡可以显示进程使用内存的方式。  若要显示[“进程属性”对话框](../debugger/process-properties-dialog-box.md)，请将焦点移动到[进程视图](../debugger/processes-view.md)窗口。  在树中选择任何进程节点，然后从“视图”菜单中选择“属性”。  
  
 “内存”选项卡上提供了下列设置：  
  
|Entry|说明|  
|-----------|--------|  
|**虚拟字节**|进程正在使用的虚拟地址空间的当前大小（以字节为单位）。  使用虚拟地址空间不一定意味着相应使用磁盘或主内存页面。  但是，虚拟空间是有限的，使用太多虚拟空间可能会限制进程加载库的能力。|  
|**峰值虚拟字节**|进程在任何时间使用的的虚拟地址空间的最大字节数。|  
|**工作集**|进程中的线程最近处理的内存页集。  如果计算机中的可用内存超过阈值，则即使页不在使用中，也会留在进程的工作集中。  如果可用内存低于阈值，将从工作集中删除页。  如果需要页，它们会在离开主内存前软故障返回到工作集中。|  
|**峰值工作集**|进程的工作集在任何时候的最大页数。|  
|**分页缓冲池字节**|进程分配的当前分页缓冲池量。  分页缓冲池是一个系统内存区域，操作系统组件在完成指定的任务后从其中获取空间。  如果系统在持续的时段内不访问分页缓冲池页，则可以将它们分页到页面文件。|  
|**非分页缓冲池字节**|进程分配的非分页缓冲池中的当前字节数。  非分页缓冲池是一个系统内存区域，操作系统组件在完成指定的任务后从其中获取空间。  非分页缓冲池页不会分页到页面文件；它们在分配时保存在主内存中。|  
|**专用字节**|进程已分配的、不能与其他进程共享的当前字节数。|  
|**可用字节**|进程的未用虚拟地址空间总量。|  
|**保留字节**|进程保留的以供将来使用的虚拟内存总量。|  
|**可用图像字节**|进程中的图像未使用或保留的虚拟地址空间量。|  
|**保留图像字节**|进程中运行的图像保留的所有虚拟内存的和。|
