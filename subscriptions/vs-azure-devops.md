# <a name="azure-devops-services-benefits-for-visual-studio-subscribers"></a>Visual Studio 订阅者的 Azure DevOps Services 权益

活动 Visual Studio 订阅者可使用订阅中包含的 Azure DevOps Services 权益的一系列功能。 可在所属的每个 Azure DevOps Services 帐户（自己创建的帐户或由其他人添加的帐户均可）中使用相同的这些功能。

使用激活 Visual Studio 订阅时所用的标识或[备用标识](vs-alternate-identity.md)登录到 Azure DevOps Services 时，会被自动识别。

[注册免费的 Azure DevOps Services 帐户](https://visualstudio.microsoft.com/team-services/)

## <a name="eligibility"></a>资格

| 订阅级别                                                 |     信道                                            | 好处                                                          | 是否续订？    |
|--------------------------------------------------------------------|---------------------------------------------------------|------------------------------------------------------------------|---------------|
| Visual Studio Enterprise（标准、年度云、月度云）   | VL、Azure、零售、所选 NFR<sup>1</sup>、FTE  | 基本功能、测试管理器、包管理、自承载 CI/CD       |  是          |
| Visual Studio Professional（标准、年度云、月度云） | VL、Azure、零售、                                        | Basic                                                            |  是          |
| Visual Studio Test Professional（标准）                         | VL、零售                                              | 基本功能、测试管理器                                              |  是          |
| MSDN 平台（标准）                                          | VL、零售                                              | 基本功能、测试管理器                                              |  是          |

<sup>1</sup> 包括：限制转售 (NFR)、Microsoft 重要合作伙伴 (MVP)、区域总监 (RD)、Visual Studio 行业合作伙伴 (VSIP)、Microsoft 合作伙伴网络 (Enterprise)、BizSpark、MCT 软件和服务开发人员。不包括：MCT 软件和服务、Imagine。

详细了解 Azure DevOps Services 功能：

- [基本](https://visualstudio.microsoft.com/team-services/compare-features/)
- [测试管理器](https://marketplace.visualstudio.com/items?itemName=ms.vss-testmanager-web)
- [包管理](https://marketplace.visualstudio.com/items?itemName=ms.feed)

无法确定正在使用哪些订阅？  连接到 [https://my.visualstudio.com/subscriptions](https://my.visualstudio.com/subscriptions?wt.mc_id=o~msft~docs)，查看分配给自己的所有订阅。
如果没有看到所有订阅，则可能是有一个或多个订阅分配给了不同的电子邮件地址。  你需要使用其他电子邮件地址登录来查看那些订阅。
